# Guía Esencial para el Mantenimiento de su Sistema Linux (Ubuntu y Pop!_OS)

El mantenimiento regular de su sistema Linux es clave para su estabilidad, seguridad y rendimiento. Esta guía simplificada le mostrará cómo mantener su software al día, gestionar sus aplicaciones y controlar las cuentas de usuario, utilizando tanto la interfaz gráfica (GUI) como la línea de comandos (CLI).


## 4. Actualización de Software Instalado

Actualizar el software es un proceso de dos pasos: primero, obtener la lista de lo nuevo, y luego, instalarlo.

#### Método Gráfico (GUI)

*   **Para Ubuntu:**
    1.  Abra el menú de aplicaciones (a menudo llamado "Actividades") y busque "Actualizador de Software" (Software Updater).
    2.  Haga clic en el icono del "Actualizador de Software". El sistema automáticamente buscará nuevas versiones de software.
    3.  Si se encuentran actualizaciones, aparecerá una ventana preguntando si desea instalarlas. Haga clic en "Instalar Ahora".
    4.  Se le pedirá su contraseña de usuario para autorizar la instalación. Introdúzcala y presione Enter.
    5.  Opcionalmente, puede hacer clic en "Detalles de las actualizaciones" para seleccionar qué software específico desea actualizar.
    6.  Una vez completada la instalación, el sistema podría indicarle que es necesario reiniciar.

*   **Para Pop!_OS:**
    1.  Abra la aplicación "Pop!_Shop" desde su menú de aplicaciones.
    2.  Las actualizaciones del sistema disponibles suelen presentarse dentro de la interfaz de Pop!_Shop, a menudo en una sección dedicada a "Actualizaciones" o "Instalado".
    3.  Inicie el proceso de actualización desde allí.
    4.  Tenga en cuenta que Pop!_OS no siempre elige el servidor de actualización más rápido. Para optimizar la velocidad, puede revisar y ajustar la configuración de los espejos en Pop!_Shop.

#### Método de Línea de Comandos (CLI)

Abra la Terminal (`Ctrl + Alt + T`).

1.  **Actualizar listas de paquetes:**
    ```bash
    sudo apt update
    ```
    Este comando descarga la información más reciente de los paquetes disponibles.
2.  **Instalar actualizaciones:**
    ```bash
    sudo apt upgrade
    ```
    Este comando instala las nuevas versiones de todo el software instalado. Puede añadir `-y` para confirmar automáticamente.
    Si ve un error como "E:dpkg was interrupted...", ejecute:
    ```bash
    sudo dpkg --configure -a
    ```
    Esto repara la base de datos de paquetes.

### 4.1. Actualización a una Versión Más Reciente del SO

Esto es un "salto de versión" (ej. de Ubuntu 22.04 a 24.04). **¡Haga una copia de seguridad completa antes!**

#### Método Gráfico (GUI)

*   **Para Ubuntu:**
    1.  Primero, asegúrese de que todo el software actualmente instalado esté completamente al día ejecutando `sudo apt update` y `sudo apt upgrade` en la terminal.
    2.  Abra la aplicación "Actualizador de Software" como lo haría para las actualizaciones regulares.
    3.  Si hay una nueva versión disponible y habilitada para su sistema, el Actualizador de Software lo notificará y ofrecerá la opción de "Actualizar...". Haga clic en este botón.
    4.  Se le presentará un enlace a las notas de la versión. Haga clic en "Actualizar" para continuar.
    5.  Siga las indicaciones en pantalla, que le guiarán a través del proceso, incluyendo la deshabilitación temporal de la pantalla de bloqueo y la aprobación de la eliminación de paquetes obsoletos.
    6.  Finalmente, se le solicitará reiniciar el sistema para completar la actualización. Haga clic en "Reiniciar Ahora".

*   **Para Pop!_OS:**
    1.  Primero, asegúrese de que el sistema haya aplicado todas las actualizaciones de software actuales ejecutando `sudo apt update` y `sudo apt full-upgrade` en la terminal.
    2.  Después de aplicar las actualizaciones, debería aparecer una notificación en la parte superior de la pantalla indicando que hay una actualización disponible.
    3.  Alternativamente, puede navegar a "Configuración -> Actualización y Recuperación del SO" (Settings -> OS Upgrade & Recovery).
    4.  En la página de actualización, se mostrará un mensaje indicando que una nueva versión de Pop!_OS está disponible con un botón de "Descargar". Haga clic en "Descargar".
    5.  Una vez completada la descarga, recibirá una segunda notificación indicando que la actualización está lista.
    6.  Haga clic en la notificación y su computadora se reiniciará a la pantalla de actualización para completar el proceso.

#### Método de Línea de Comandos (CLI)

*   **Para Ubuntu:**
    1.  Actualice el sistema de forma normal:
        ```bash
        sudo apt update && sudo apt dist-upgrade -y
        ```
        `dist-upgrade` es clave para actualizaciones de versión.
    2.  Inicie la actualización de versión:
        ```bash
        sudo do-release-upgrade
        ```
        Siga las indicaciones. Para versiones de desarrollo (no estables), use `sudo do-release-upgrade -d`.
*   **Para Pop!_OS:**
    1.  Actualice el sistema:
        ```bash
        sudo apt update && sudo apt full-upgrade
        ```
    2.  Inicie la actualización de versión:
        ```bash
        pop-upgrade release upgrade
        ```
        Pop!_OS tiene herramientas propias que interactúan con su partición de recuperación.

## 5. Gestión de Su Software: Eliminación

Eliminar software puede ser simple o muy exhaustivo, dependiendo de si quiere dejar rastros o no.

### 5.1. Eliminación de Software

#### Método Gráfico (GUI)

*   **Para Ubuntu:**
    1.  Abra el "Centro de Software" o "Ubuntu Software" desde su menú de aplicaciones.
    2.  Haga clic en la pestaña "Instalado" para ver una lista del software actualmente instalado.
    3.  Busque el programa que desea desinstalar o desplácese por la lista hasta encontrarlo.
    4.  Haga clic en el botón "Eliminar" que se encuentra junto al programa.
    5.  Si se le solicita confirmación, haga clic en "Eliminar" nuevamente o en "Aceptar".

*   **Para Pop!_OS:**
    1.  Abra la aplicación "Pop!_Shop" desde su menú de aplicaciones.
    2.  Busque la aplicación que desea desinstalar.
    3.  Si la aplicación está instalada, debería aparecer una opción para desinstalarla en su página o en el listado. Haga clic en esta opción.
    4.  Siga las indicaciones en pantalla para completar la desinstalación.

**Nota:** Las herramientas gráficas a menudo realizan una eliminación básica que puede dejar archivos de configuración y dependencias no utilizadas.

#### Método de Línea de Comandos (CLI)

Abra la Terminal.

*   **Encontrar nombre del paquete:**
    *   Para paquetes DEB: `dpkg -l | grep <nombre_parcial>` o `sudo apt list --installed`.
    *   Para paquetes Snap: `snap list`.
*   **Desinstalar (conserva archivos de configuración):**
    ```bash
    sudo apt remove <nombre_del_paquete>
    ```
    Útil si planea reinstalar.
*   **Eliminar completamente (incluye archivos de configuración):**
    ```bash
    sudo apt purge <nombre_del_paquete>
    ```
    Esto limpia todo rastro.
*   **Eliminar dependencias no usadas:**
    ```bash
    sudo apt autoremove
    ```
    Limpia paquetes que ya no son necesarios.
*   **Limpiar caché de paquetes descargados:**
    ```bash
    sudo apt clean
    ```
    Libera espacio en disco.
*   **Desinstalar paquetes Snap:**
    ```bash
    sudo snap remove <nombre_del_paquete>
    ```
*   **Desinstalar paquetes `.deb` instalados directamente:**
    ```bash
    sudo dpkg -r <nombre_del_paquete>
    ```
    Solo elimina el paquete, no sus dependencias.
*   **Para software de PPAs:** Instale `ppa-purge` (`sudo apt install ppa-purge`) y luego use `sudo ppa-purge <ppa:nombre_del_repositorio>` para eliminar el software y el PPA.

### 5.2. Eliminación de Archivos y Directorios

La eliminación de archivos puede ser temporal (a la papelera) o permanente.

#### Método Gráfico (GUI)

*   **Para Ubuntu (y general para la mayoría de entornos de escritorio Linux):**
    1.  Abra su administrador de archivos (por ejemplo, "Archivos" o Nautilus).
    2.  Navegue hasta la ubicación del archivo o directorio que desea eliminar.
    3.  Haga clic derecho sobre el archivo o directorio.
    4.  Seleccione "Mover a la Papelera" o "Eliminar" en el menú contextual.
    5.  Los elementos movidos a la papelera pueden ser recuperados posteriormente si es necesario. Para una eliminación permanente, puede vaciar la papelera.

*   **Para Pop!_OS (Cosmic Files):**
    1.  Abra el administrador de archivos "Cosmic Files".
    2.  Navegue hasta el archivo o directorio que desea eliminar.
    3.  Para mover a la papelera: Haga clic derecho sobre el archivo/directorio y seleccione "Mover a la Papelera" o "Eliminar".
    4.  Para eliminar permanentemente (sin pasar por la papelera): Seleccione el archivo o directorio y presione `Shift + Supr`. Se le debería mostrar una ventana de confirmación preguntando: "¿Seguro que quiere eliminar esto completamente?". Confirme para proceder.

#### Método de Línea de Comandos (CLI)

La eliminación por CLI es permanente y no va a la papelera.

*   **Eliminar archivo:**
    ```bash
    rm <nombre_de_archivo>
    ```
    Ej: `rm notas.txt`. Use `rm -f` para forzar sin confirmación (¡cuidado!).
*   **Eliminar directorio vacío:**
    ```bash
    rmdir <nombre_de_directorio_vacio>
    ```
    O `rm -d <nombre_de_directorio_vacio>`.
*   **Eliminar directorio no vacío (y su contenido):**
    ```bash
    rm -r <nombre_de_directorio>
    ```
    Esto es recursivo.
*   **Eliminar directorio no vacío (y su contenido, forzado):**
    ```bash
    rm -rf <nombre_de_directorio>
    ```
    **¡EXTREMA PRECAUCIÓN!** Este comando es irreversible y no pide confirmación.
*   **Eliminación segura (sobrescribe datos):**
    ```bash
    shred [opciones] <nombre_de_archivo>
    ```
    Puede no ser efectivo en SSDs o sistemas de archivos modernos.
*   **Eliminación segura (a la papelera CLI):** Instale `trash-cli` (`sudo apt install trash-cli`) y use:
    ```bash
    trash-put <nombre_de_archivo>
    ```
    Esto mueve los archivos a la papelera del sistema.

**Consejo:** Antes de usar `rm` con comodines (`*`, `?`), use `ls` con el mismo patrón para ver qué se eliminará.

## 6. Control de Cuentas: Gestión de Usuarios y Grupos

La gestión de usuarios y grupos es vital para la seguridad y organización de su sistema.

### 6.1. Gestión de Cuentas de Usuario

#### Método Gráfico (GUI)

*   **Para Ubuntu y Pop!_OS (usando "Configuración -> Usuarios"):**
    1.  Abra la aplicación "Configuración" desde su menú de aplicaciones.
    2.  Desplácese por la lista de opciones y seleccione la pestaña "Usuarios".
    3.  Haga clic en el botón "Desbloquear" (generalmente en la esquina superior derecha) e introduzca su contraseña de usuario para permitir cambios.
    4.  Haga clic en el botón "Añadir Usuario" para abrir un nuevo cuadro de diálogo.
    5.  Introduzca el nombre completo del nuevo usuario. El sistema sugerirá un nombre de usuario, que puede modificar si lo desea.
    6.  Puede establecer una contraseña para el usuario en este momento o permitir que el usuario la establezca en su primer inicio de sesión.
    7.  Opcionalmente, puede otorgar privilegios de administrador al nuevo usuario haciendo clic en la pestaña "Administrador" durante la creación o activando el botón de administrador si la cuenta ya existe.
    8.  Haga clic en "Añadir" para guardar la nueva cuenta. Se le solicitará su contraseña nuevamente para confirmar la creación.
    9.  Para eliminar un usuario: Siga los pasos 1-3 para desbloquear la configuración de usuarios.
    10. Seleccione la cuenta de usuario que desea eliminar y haga clic en el botón "Eliminar usuario".
    11. Si el usuario está conectado, recibirá una advertencia. Puede elegir "Eliminar Archivos" para borrar el directorio personal del usuario o "Mantener Archivos" si desea conservar sus datos.

#### Método de Línea de Comandos (CLI)

Abra la Terminal.

*   **Añadir usuario:**
    ```bash
    sudo adduser <nombre_de_usuario>
    ```
    Se le pedirá una contraseña y otros detalles.
*   **Eliminar usuario:**
    ```bash
    sudo deluser <nombre_de_usuario>
    ```
    Esto elimina la cuenta y su grupo primario, pero no el directorio personal.
    Para eliminar también el directorio personal:
    ```bash
    sudo deluser --remove-home <nombre_de_usuario>
    ```
*   **Bloquear/Desbloquear contraseña:**
    *   Bloquear: `sudo passwd -l <nombre_de_usuario>`
    *   Desbloquear: `sudo passwd -u <nombre_de_usuario>`
*   **`sudo` vs. `root`:** Ubuntu y Pop!_OS promueven el uso de `sudo` para tareas administrativas, que permite elevar privilegios temporalmente con su propia contraseña, en lugar de iniciar sesión directamente como `root`.

### 6.2. Gestión de Grupos

Los grupos simplifican la gestión de permisos para múltiples usuarios.

#### Método Gráfico (GUI)

*   **Para Ubuntu (usando "Usuarios y Grupos"):**
    1.  Para acceder a una herramienta GUI más completa para la gestión de grupos, puede que necesite instalar el paquete `gnome-system-tools`. Abra la Terminal y ejecute:
        ```bash
        sudo apt install gnome-system-tools
        ```
    2.  Una vez instalado, busque "Usuarios y Grupos" en su menú de aplicaciones (o ejecute `users-admin` en la Terminal).
    3.  Esta utilidad le permitirá gestionar usuarios y sus membresías en grupos, incluyendo la adición de usuarios a grupos como `sudo` para permisos de administrador.

*   **Para Pop!_OS:**
    1.  Las herramientas GUI integradas en Pop!_OS (como "Configuración -> Usuarios") se centran principalmente en la gestión de usuarios y la adición de usuarios a grupos *existentes*.
    2.  La creación directa de nuevos grupos desde cero a través de una interfaz gráfica puede ser limitada o menos intuitiva en comparación con la línea de comandos. Para una gestión integral de grupos, especialmente para la creación de grupos personalizados, la CLI suele ser el método más robusto y directo.

#### Método de Línea de Comandos (CLI)

Abra la Terminal.

*   **Añadir grupo:**
    ```bash
    sudo addgroup <nombre_de_grupo>
    ```
    Ej: `sudo addgroup desarrolladores`.
*   **Eliminar grupo:**
    ```bash
    sudo delgroup <nombre_de_grupo>
    ```
    Ej: `sudo delgroup desarrolladores`.
*   **Añadir usuario a un grupo:**
    ```bash
    sudo adduser <nombre_de_usuario> <nombre_de_grupo>
    ```
    O, más comúnmente para grupos suplementarios:
    ```bash
    sudo usermod -aG <nombre_de_grupo> <nombre_de_usuario>
    ```
    Ej: `sudo usermod -aG sudo miusuario` para añadir a `sudo`.
*   **Listar grupos:**
    *   `cat /etc/group`
    *   `getent group`
    *   `groups <nombre_de_usuario>` (para ver los grupos de un usuario específico)

## Conclusión

Dominar las actualizaciones, la eliminación de software y la gestión de cuentas en Ubuntu o Pop!_OS le da un control total sobre su sistema. Ya sea que prefiera la comodidad visual de la GUI o la precisión de la CLI, ambos métodos son herramientas poderosas. Recuerde siempre la regla de oro: **¡haga copias de seguridad regularmente!** Es su mejor defensa contra cualquier error.
